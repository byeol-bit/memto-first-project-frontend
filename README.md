## 📖 개요

### 🚀 프로젝트 시작 계기
> "우리의 기억을 더 가치 있게 기록할 수는 없을까?"  
`Memto`는 단순한 기록을 넘어, 사용자간의 소중한 순간을 공유하고 보관하기 위해 시작되었습니다. 첫 프로젝트인 만큼 프론트엔드의 기초부터 탄탄히 다지는 것을 목표로 했습니다.

### 📅 초기 플랜
- **Phase 1:** 요구사항 정의 및 와이어프레임 설계
- **Phase 2:** 핵심 UI 컴포넌트 개발 및 상태 관리 라이브러리 도입
- **Phase 3:** API 연동 및 사용자 경험(UX) 최적화

| 이름 | 역할 | 담당 부분 |
| :--- | :---: | :--- |
| **팀원1** | FE Lead | 프로젝트 아키텍처 설계, 메인 페이지, 인증 로직 |

## ⚙️ 개발 과정

### ⚡ 진행 과정 상세
1. **기획 단계:** Figma를 활용한 상세 디자인 및 API 명세서 작성
2. **개발 단계:** Atomic Design 패턴을 적용한 컴포넌트화 작업
3. **트러블슈팅:** API 통신 중 발생한 CORS 이슈 해결 및 성능 최적화

### 🏃 수행 전 팀원 상태
> [!NOTE]
> 프로젝트 시작 전, 저희 팀은 **"이론은 알지만 실전은 처음"**인 상태였습니다.
> - **팀원1:** React Hooks 활용 능력 중급, 협업 툴(Git) 경험 다수
> - **팀원2:** HTML/CSS 마크업 익숙, React 프레임워크 첫 도전
> - **팀원3:** JavaScript 기초 탄탄, 상태 관리(Redux/Recoil) 개념 학습 중

## 📈 성장 및 회고

### 🏆 수행 후 발전 사항
- **기술적 성장:** 단순 기능 구현을 넘어 **재사용 가능한 컴포넌트 설계**의 중요성을 깨달았습니다.
- **협업 능력:** Git Flow 전략을 도입하여 충돌 없이 효율적으로 코드를 관리하는 법을 익혔습니다.

### ⚠️ 아쉬웠던 점
> [!IMPORTANT]
> **성능 최적화에 대한 미련** > 초기 플랜에서 계획했던 'Lighthouse 점수 90점 이상'을 달성하지 못한 점이 아쉽습니다. 이미지 최적화 및 코드 분할(Code Splitting)에 대해 더 깊이 공부할 필요를 느꼈습니다.
